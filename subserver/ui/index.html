客户端下载

SubServer 提供 Clash.Meta / mihomo 格式的订阅配置。请根据您的平台选择合适的客户端：

平台客户端下载地址说明
WindowsClash Verge RevGitHub Releases推荐，支持 mihomo 内核
Windowsv2rayNGitHub Releases支持 VLESS/VMess/Reality
macOSClash Verge RevGitHub Releases推荐，跨平台
macOSClashX MetaGitHub Releases原生 macOS 客户端
LinuxClash Verge RevGitHub Releases推荐，GUI 客户端
Linuxmihomo (CLI)GitHub Releases命令行，需手动配置
iOSShadowrocketApp Store 搜索付费，支持订阅导入

导入订阅

  1. 在「我的订阅」页面复制基础订阅地址或完整配置导出URL
  2. 打开客户端，找到「订阅」或「Profiles」设置
  3. 粘贴订阅 URL，点击「更新」或「Import」
  4. 选择导入的配置作为活动配置
  5. 选择节点并连接
基础订阅地址仅包含节点列表（Proxies YAML），需配合模板使用。完整配置导出可直接导入 Clash.Meta 客户端使用。

Windows

系统全局代理（HTTP）

  1. 打开 设置 → 网络和 Internet → 代理
  2. 在「手动设置代理」中开启
  3. 地址填入 127.0.0.1，端口填入客户端混合端口（通常 7890）
  4. 点击保存
Windows 系统代理仅支持 HTTP，不支持 SOCKS5 认证。建议使用浏览器插件或客户端内置的 TUN 模式。

浏览器代理（SwitchyOmega）

  1. 安装 Proxy SwitchyOmega 扩展
  2. 新建情景模式，选择代理服务器
  3. 协议选 SOCKS5 或 HTTP，地址 127.0.0.1，端口填客户端端口
  4. 点击「应用选项」

命令行

# PowerShell — 设置 HTTP 代理（当前会话）
$env:HTTP_PROXY = "http://127.0.0.1:7890"
$env:HTTPS_PROXY = "http://127.0.0.1:7890"

# CMD
set HTTP_PROXY=http://127.0.0.1:7890
set HTTPS_PROXY=http://127.0.0.1:7890

macOS

系统全局代理

  1. 打开 系统设置 → 网络 → 选择当前网络 → 详细信息 → 代理
  2. SOCKS 代理：地址 127.0.0.1，端口填客户端 SOCKS 端口
  3. Web 代理（HTTP）：地址 127.0.0.1，端口填客户端 HTTP 端口

终端

# 在 ~/.zshrc 中添加（永久生效）：
export http_proxy="http://127.0.0.1:7890"
export https_proxy="http://127.0.0.1:7890"
export all_proxy="socks5://127.0.0.1:7890"

# 临时使用（仅当前终端）：
export ALL_PROXY="socks5h://127.0.0.1:7890"
socks5h:// 表示 DNS 也通过代理，推荐使用。

Linux

环境变量

# 在 ~/.bashrc 或 ~/.profile 中添加：
export http_proxy="http://127.0.0.1:7890"
export https_proxy="http://127.0.0.1:7890"
export all_proxy="socks5://127.0.0.1:7890"
export no_proxy="localhost,127.0.0.1,::1,10.0.0.0/8,192.168.0.0/16"

Git

git config --global http.proxy "socks5://127.0.0.1:7890"
git config --global https.proxy "socks5://127.0.0.1:7890"

# 取消代理：
git config --global --unset http.proxy
git config --global --unset https.proxy

iOS

Shadowrocket（推荐）

  1. 打开 Shadowrocket → 点击右上角 +
  2. 类型选择 Subscribe
  3. URL 填入订阅地址
  4. 保存并更新

WiFi 代理（HTTP）

  1. 打开 设置 → WiFi → 点击已连接网络旁的 (i)
  2. 滚动到最下方 → HTTP 代理 → 选择 手动
  3. 服务器填入 127.0.0.1，端口填客户端 HTTP 端口
iOS 原生不支持 SOCKS5 代理。如需 SOCKS5 或 VLESS，请使用 Shadowrocket 等第三方客户端。

Android

Clash Meta for Android（推荐）

  1. 打开应用 → 配置 → 新增配置
  2. 选择 URL 方式
  3. 填入订阅地址并更新
  4. 返回主界面，选择配置并启动

WiFi 代理（HTTP）

  1. 打开 设置 → WiFi → 长按已连接的网络 → 修改网络
  2. 展开 高级选项 → 代理选择 手动
  3. 代理主机 127.0.0.1，端口填客户端端口

curl / wget

# curl 通过 SOCKS5
curl -x socks5h://127.0.0.1:7890 https://example.com

# curl 通过 HTTP
curl -x http://127.0.0.1:7890 https://example.com

# wget
wget -e use_proxy=yes \
     -e http_proxy="http://127.0.0.1:7890" \
     -e https_proxy="http://127.0.0.1:7890" \
     https://example.com

npm / pip / Go

# npm
npm config set proxy http://127.0.0.1:7890
npm config set https-proxy http://127.0.0.1:7890
# 取消：npm config delete proxy && npm config delete https-proxy

# pip
pip install --proxy http://127.0.0.1:7890 包名

# Go
export HTTP_PROXY="http://127.0.0.1:7890"
export HTTPS_PROXY="http://127.0.0.1:7890"
export GOPROXY="https://proxy.golang.org,direct"

Docker

# ~/.docker/config.json
{
  "proxies": {
    "default": {
      "httpProxy": "http://127.0.0.1:7890",
      "httpsProxy": "http://127.0.0.1:7890",
      "noProxy": "localhost,127.0.0.1"
    }
  }
}

SSH（通过 SOCKS5 跳板）

ssh -o ProxyCommand='nc -x 127.0.0.1:7890 -X 5 %h %p' user@target-host

proxychains（强制任意应用走代理）

# 安装 (macOS)
brew install proxychains-ng
# 安装 (Linux)
apt install proxychains4

# 配置 ~/.proxychains/proxychains.conf 末尾添加：
socks5  127.0.0.1  7890

# 使用
proxychains4 curl https://www.google.com
proxychains4 npm install

常见问题

Q: 基础订阅和完整配置导出有什么区别？

基础订阅仅返回节点列表（proxies YAML 段），需要配合模板使用或手动粘贴到配置文件中。完整配置导出包含完整的 Clash.Meta 配置（包括代理组、规则等），可直接导入客户端使用。

Q: HTTP 代理和 SOCKS5 代理有什么区别？

HTTP 代理工作在应用层，只代理 HTTP/HTTPS 流量，兼容性最好。SOCKS5 代理工作在传输层，可代理所有 TCP/UDP 流量（包括游戏、DNS 等），功能更强。

Q: 设置了代理后无法连接？

请逐步检查： 1. 客户端是否正在运行且已选择节点 2. 端口是否正确（通常 7890） 3. 防火墙是否允许本地连接 4. 节点本身是否可用

Q: 什么是 VLESS+Reality？

VLESS+Reality 是目前最强的抗审查代理协议，流量伪装为访问合法网站（如 microsoft.com），极难被检测和封锁。需要使用支持该协议的客户端。

Q: 如何取消代理设置？

删除或注释掉对应的环境变量，或在系统/浏览器设置中关闭代理。重启终端或浏览器使更改生效。

Q: socks5:// 和 socks5h:// 有什么区别？

socks5:// 表示本地解析 DNS 后通过代理连接；socks5h:// 表示 DNS 也通过代理解析，后者更安全，推荐使用。

Q: 订阅地址无法更新？

请检查： 1. 订阅 URL 是否完整复制 2. 网络是否能访问订阅服务器 3. 账号是否已启用 4. 联系管理员确认节点配置是否正常
